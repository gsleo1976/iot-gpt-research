## What This Project Is

IoT-GPT is a research project by Gurdev Singh (gurdev.leo@gmail.com) building an intelligent multi-device coordination system for sustainable smart homes. Core ideas:
- **Device Attribute Registry (DAR):** JSON-LD knowledge base describing every connected device
- **Multi-Device Coordination Engine:** schedules devices to avoid simultaneous peak loads
- **Calm Technology Layer:** ambient coloured light signals (smart bulbs) instead of phone notifications
- Runs fully on-device on a Raspberry Pi 5, no cloud required
- Colour palette: BG `#0D3B3B`, accent `#F4A261` (orange), CARD `#0A4A4A`, MINT `#2EC4B6`, SKY `#48CAE4`

## Known Issues / Watch Out For

- `CT_TextBody has no get_or_add_bodyPr` — use `tf._txBody.find(qn('a:bodyPr'))` with lxml fallback
- `'Shape' object has no attribute 'shape'` — use `sp._element.spPr` not `sp.shape._element.spPr`
- Slide removal `KeyError: None` — use full namespace `{http://schemas.openxmlformats.org/officeDocument/2006/relationships}id`
- `rm` on mnt/iot files gets `Operation not permitted` — use `mcp__cowork__allow_cowork_file_delete` tool first
- `pip install` on Pi OS: always add `--break-system-packages`
- `soffice` PDF conversion: needs `--outdir /sessions/...` flag

## 6 Experiments in the Research

1. **Energy Savings from Coordinated Scheduling** — IoT-GPT vs. default device schedules
2. **Calm vs. Push Notification User Study** — 20 participants, crossover, NASA-TLX
3. **IoT-GPT vs. Manual Device Management** — 15 participants as "expert home managers"
4. **Multi-Device Load Optimisation Under Constraints** — 5 constraint scenarios
5. **Ambient Snowfall Alert Accuracy** — 60-day field study in 2 pilot households
6. **Scalability — 3 to 15 Devices** — schedule generation time vs. device count
